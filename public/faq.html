Basics

ShopGraph is a product data extraction API that shows which extraction method produced each field. Every URL returns structured product data with per-field confidence scoring and extraction provenance metadata. It supports B2C extraction and B2B product data extraction. Available as a REST API and MCP server.

ShopGraph extracts: product title, price, currency, brand, description, images, availability, SKU, GTIN/UPC, manufacturer part number, category, ratings, reviews, variants, and B2B fields (MOQ, lead time, bulk pricing). See the full ProductData schema.

Every field includes a confidence score from 0 to 1. The score starts with a baseline from the extraction method (Schema.org: 0.93, LLM: 0.70 base) then adjusted per field by modifiers (cross-validation, format checks, etc.). You can set a strict_confidence_threshold to filter out low-confidence fields. See Confidence Scoring for details.

UCP (Universal Commerce Protocol) is a standard schema for commerce data interchange. When you add format=ucp to your request, ShopGraph returns data in UCP line_item format, validated against ucp-schema v1.1.0. See UCP Output.

AgentReady scoring evaluates extracted product data across six dimensions — completeness (structured_data_completeness), semantic richness (semantic_richness), UCP compatibility (ucp_compatibility), pricing clarity (pricing_clarity), inventory signals (inventory_signal_quality), and access readiness (access_readiness) — with 5 currently active (access_readiness stubbed at weight 0.00) to produce a single 0-100 score indicating how suitable the data is for autonomous agent workflows. See AgentReady Scoring.

Anyone whose automation writes product data into a downstream system: procurement agents populating purchase orders (procurement automation), price monitoring pipelines updating dashboards (price monitoring API), catalog sync jobs feeding marketplace listings (catalog sync API), competitive intelligence tools comparing specifications (competitive intelligence API), product comparison engines aggregating across retailers, supply chain visibility platforms tracking supplier inventory (supply chain product data, inventory management API), and marketplace listing automation services normalizing product data across channels.

ShopGraph uses a three-tier extraction pipeline. Tier 1 reads Schema.org structured markup — data the merchant placed intentionally. Tier 2 uses an LLM to interpret page text. Tier 3 executes JavaScript in a headless browser, then interprets the rendered result. Tiers 2 and 3 involve inference. When a field's value comes only from inference (no structured markup confirmation), the confidence score reflects that. A "guess" is any field where the extraction method required interpretation rather than reading explicit structured data.

Three things. First, scores are calibrated against a 208-URL ground-truth dataset across 22 product verticals. Second, cross-tier agreement is built into the score: when two extraction methods agree and one dissents, the confidence reflects the disagreement, not just the majority. Third, every field includes its extraction provenance — you can see which method produced the value and verify it yourself. The scores are not a black box opinion. Honest limitation: confidence is only as good as the calibration dataset. We publish the methodology.

A product page can have a price extracted from Schema.org markup (0.93 confidence) and an availability status inferred from body text (0.55 confidence). A single document-level score averages these into something meaningless. Per-field confidence scoring preserves the actionable signal: auto-fill the price, route the availability to human review.

Extraction provenance. Those tools extract product data (some very well). ShopGraph also extracts product data, but returns the extraction method and confidence for every field. No incumbent offers per-field confidence on product extraction, cross-tier validation, or server-side threshold enforcement (strict_confidence_threshold). If your workflow just needs JSON from a product URL, any extractor works. If your workflow needs to know which fields are reliable enough for autonomous action — for example, marketplace listing automation or price monitoring — that is what ShopGraph adds.

Usage

The easiest way is to try the Playground at shopgraph.dev — no signup required. Or call POST /api/enrich with a product URL. The full pipeline is available on the free tier (50 calls/month). See the installation guide for all integration options.

ShopGraph works with any product page on the public web. It has been tested across 22 verticals covering both B2C (Shopify, Amazon, Best Buy, etc.) and B2B (Uline, Haas Tooling, AM Leonard, etc.) across a 208-URL ground-truth dataset.

Latency varies by endpoint, site complexity, and cache state. Cached responses return in under 200ms. Full extraction times depend on whether the site requires JavaScript rendering or RFC 9421 identity signing.

Because not every product page has Schema.org markup. Tier 1 (structured data) is the most reliable, but coverage varies: some merchants mark up price and name but not availability or specifications. Tier 2 (LLM) extracts fields that structured markup misses. The key is that the confidence score reflects which tier produced the value. Tier 3 executes the JavaScript and extracts from the rendered DOM. This is the most expensive tier (slowest, most resource-intensive) and ShopGraph only routes to it when Tiers 1 and 2 cannot produce the field. The confidence score for browser-extracted fields reflects the additional inference step.

Pricing

Four tiers: Free (50/mo, $0), Starter (10K/mo, $99), Growth (50K/mo, $299), Enterprise (custom). Agents can also pay per-call via Stripe MPP without a subscription. See Pricing.

Stripe Machine-to-Machine Payments (MPP) lets AI agents pay per API call without a pre-purchased subscription. The agent passes a payment_method_id with each request and is charged $0.02 per full extraction.

Each POST to /api/enrich, /api/enrich/basic, or /api/enrich/html counts as one call. Cached responses (within TTL) do not count against your quota. Scoring-only requests (/api/score) are counted separately.

The API returns HTTP 429 with a Retry-After header indicating when you can send the next request. Cached responses (within TTL) do not count against your quota. If you consistently hit limits, upgrade your tier or contact us about Enterprise plans.
