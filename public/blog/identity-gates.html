The Open Web Got Identity Gates. What Commerce AI Agents Need Now.

Krishna Brown · April 12, 2026

On April 7th, Cloudflare and GoDaddy announced a partnership to build identity infrastructure for AI agents on the open web. Web Bot Auth. Agent Networking Standard. RFC 9421 cryptographic signatures. The message was clear: anonymous access to the web is ending. Agents that can prove who they are get through. Agents that can't, don't.

I'd been building toward this for weeks.

What changed

ShopGraph started as an extraction tool. Pull product data from a URL, score how much to trust each field, return structured JSON. That's still what it does. But the problem underneath shifted.

When I launched in March, the extraction challenge was mostly about data quality. Could the system handle a JavaScript-rendered B2B catalog? Could it fill in missing Schema.org markup with LLM extraction? Could it score each field honestly so downstream agents knew what to trust?

By April, the challenge was also about access. CDN providers started deploying identity gates. Cloudflare's AI Bot Management. GoDaddy's Agent Networking Standard. Sites that were open last month now require cryptographic proof that you're a legitimate agent before they'll serve you data.

This is the right direction for the web. Commerce sites want agent traffic. Agents drive transactions. But sites need to distinguish trusted automation from anonymous bots. The infrastructure for making that distinction is being built right now.

What I built

ShopGraph v1.0.0 and v1.0.1 shipped in the first week of April. The build covered three layers.

The trust layer. Per-field confidence scoring went from a useful feature to the core product. Every field in every response carries a score from 0 to 1. Your agent can route decisions per-field: act autonomously on high-confidence data, flag uncertain fields for human review, reject unreliable data entirely. The system is transparent about its own uncertainty. That's the design decision everything else is built on.

The standards layer. UCP output format, validated against Google's ucp-schema v1.1.0. Stripe Machine-to-Machine Payments wired in so agents can pay per-extraction without a subscription. AgentReady scoring across six dimensions that tells your agent how suitable a target URL's data is for automated workflows before you commit pipeline resources.

The identity layer. RFC 9421 signature compliance. ShopGraph acts as a trusted proxy, handling the cryptographic handshakes that CDN security gates require. Your agents inherit ShopGraph's verified identity at the CDN layer. You don't manage proxies, keys, or identity infrastructure.

The portfolio went from a single MCP server to: a REST API, an MCP server, n8n/Make.com/Power Automate integration nodes (in testing), a public leaderboard scoring how agent-ready commerce sites are, and framework PRs for Vercel AI SDK and LangChain shipping this week.

Why B2B

The shift toward B2B industrial supply wasn't a pivot. It was following the problem to where it's hardest and most valuable.

B2C e-commerce is largely solved. Shopify stores have clean structured data. Amazon has an API. The extraction pipeline handles these sites on Tier 1 in under a second.

B2B procurement is different. A facilities manager at a hospital network needs pricing on industrial pumps from Grainger, air handling units from McMaster-Carr, electrical components from Digi-Key. Each supplier portal has a different data architecture. Some have structured markup. Many don't. Some require JavaScript rendering. Some are behind CDN gates.

The procurement teams that need reliable product data the most are working with suppliers whose sites are the hardest to extract from. And these are high-value transactions. A wrong price on an industrial component order isn't a $20 mistake. It's a $20,000 mistake.

ShopGraph's test corpus is 44% B2B, 56% B2C. The scoring system rewards B2B-specific fields: MOQ, lead time, bulk pricing tiers, manufacturer part numbers. The leaderboard tracks Grainger, McMaster-Carr, MSC Industrial, Fastenal, Digi-Key, and Mouser alongside the consumer sites.

What I got wrong

The initial site copy was fear-based. "Stop acting on probabilistic guesses." "Don't let your agent get blocked." Every feature section opened with what the developer should be afraid of instead of what ShopGraph enables. Technical builders see through that immediately. Capability-forward descriptions are more credible and more useful. I rewrote every page.

The MCP-only launch produced zero organic usage despite listings on every major directory. Directories confirm you exist. They don't create demand. The playground, the free API tier (500 calls, no signup), and the framework PRs are what create the evaluation path.

I also underestimated how much documentation consistency matters when you're building in public. Three pages on the site used different names for the same scoring dimensions. The API response example showed a different schema than the methodology page described. Framework reviewers will test your docs against your API. If those don't match, you've lost credibility before they've written a line of code.

What's next

The near-term build is distribution. Framework PRs for Vercel AI SDK and LangChain submit this week. These aren't just code contributions. They're the first time most developers will see confidence-aware extraction in a working example. An agent that checks the confidence score on every field and routes accordingly. That pattern doesn't exist in any framework cookbook yet.

The medium-term build is the access readiness dimension. ShopGraph already tracks whether target URLs require agent authentication, but the dimension is stubbed at weight 0.00 in the scoring system. It activates when Web Bot Auth adoption reaches detection threshold. When more than 10% of the test corpus requires cryptographic identity, access readiness becomes a scored dimension and the weights redistribute.

The longer-term build is CommerceFlow, a multi-step agent orchestration layer where extraction, scoring, and purchasing happen in a coordinated workflow. Stripe MPP handles the payment. ShopGraph handles the data and identity. The agent handles the decision logic. The human handles the exceptions.

The boundary between what the agent handles and what the human handles is the design problem I keep coming back to. Confidence scoring is how you draw that boundary with data instead of guessing.
