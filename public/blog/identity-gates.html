Commerce Sites Are Adding Agent Identity Verification. What It Means for Your Pipeline.

Krishna Brown · April 12, 2026

On April 7th, Cloudflare and GoDaddy announced a partnership to build identity infrastructure for AI agents on the open web. Web Bot Auth. Agent Networking Standard. RFC 9421 cryptographic signatures. The message was clear: anonymous access to the web is ending. Agents that can prove who they are get through. Agents that cannot get blocked.

This matters for anyone building agents that need third-party data. If your agent calls an external API, reads a product page, or pulls pricing from a supplier portal, the rules just changed. The sites your agent accessed last month may now require cryptographic proof of identity before serving a response.

ShopGraph handles the identity handshake. Your agent gets scored data.

I built ShopGraph to solve this exact problem. It is an authenticated product data extraction service. You send a URL. ShopGraph handles the identity negotiation with the CDN layer, extracts the product data, scores every field for confidence, and returns structured JSON. Your agent never touches the identity infrastructure.

What changed

When ShopGraph launched in March, the extraction challenge was mostly about data quality. Could the system handle a JavaScript-rendered B2B catalog? Could it fill in missing Schema.org markup with LLM extraction? Could it score each field honestly so downstream agents knew what to trust?

By April, the challenge was also about access. CDN providers started deploying identity gates. Cloudflare's AI Bot Management. GoDaddy's Agent Networking Standard. Sites that were open last month now require cryptographic proof that you are a legitimate agent before they will serve you data. This is the right direction for the web. Commerce sites want agent traffic because agents drive transactions. But sites need to distinguish trusted automation from anonymous bots. The infrastructure for making that distinction is being built right now.

What I built

ShopGraph v1.0.0 and v1.0.1 shipped in the first week of April. The build covered three layers.

The trust layer. Per-field confidence scoring went from a useful feature to the core product. Every field in every response carries a score from 0 to 1. Your agent can route decisions per-field: act autonomously on high-confidence data, flag uncertain fields for human review, reject unreliable data entirely. The system is transparent about its own uncertainty. That is the design decision everything else is built on.

The standards layer. UCP output format, validated against Google's ucp-schema v1.1.0. Stripe Machine-to-Machine Payments wired in so agents can pay per-extraction without a subscription. AgentReady scoring across six dimensions that tells your agent how suitable a target URL's data is for automated workflows before you commit pipeline resources.

The identity layer. RFC 9421 signature compliance. ShopGraph acts as a trusted proxy, handling the cryptographic handshakes that CDN security gates require. Your agents inherit ShopGraph's verified identity at the CDN layer. You do not manage proxies, keys, or identity infrastructure.

Because ShopGraph's verified identity is the shared resource, it is also the shared liability. One bad actor abusing the proxy would burn the reputation for every legitimate user. The system enforces per-agent rate limits, monitors for usage anomalies, and isolates tenant traffic so that one integration's behavior does not degrade access for others.

The portfolio went from a single MCP server to: a REST API, an MCP server, n8n and Make.com integration nodes (in testing), a playground, a public leaderboard scoring how agent-ready commerce sites are, and framework PRs for Vercel AI SDK and LangChain shipping this week.

One engine. Multiple integration surfaces.

Why B2B

B2C e-commerce is largely solved. Shopify stores have clean structured data. Amazon has an API. The extraction pipeline handles these sites on Tier 1 in under a second.

B2B procurement is different. A facilities manager at a hospital network needs pricing on industrial packaging from Uline, tooling from Haas Tooling, educational supplies from AM Leonard, air handling units from McMaster-Carr, electrical components from Digi-Key. Each supplier portal has a different data architecture. Some have structured markup. Many do not. Some require JavaScript rendering. Some are behind CDN gates. The procurement teams that need reliable product data the most are working with suppliers whose sites are the hardest to extract from. And these are high-value transactions. A wrong price on an industrial component order is not a $20 mistake. It is a $20,000 mistake.

ShopGraph's test corpus is 44% B2B, 56% B2C. The scoring system rewards B2B-specific fields: MOQ, lead time, bulk pricing tiers, manufacturer part numbers. The leaderboard tracks Grainger, McMaster-Carr, MSC Industrial, Fastenal, Digi-Key, and Mouser alongside the consumer sites.

What I got wrong

Every system designed for a stable environment breaks when the environment shifts. I did not build the identity layer first. ShopGraph launched as a pure extraction and scoring service. The RFC 9421 proxy came later, after Cloudflare and GoDaddy announced their partnership, after sites in the test corpus started returning 403s that had returned 200s the week before. The conditions changed, and the system had to change with them. If I were starting today, the identity layer would be the foundation, not the addition.

What's next

The near-term build is distribution. Framework PRs for Vercel AI SDK and LangChain submit this week. These are not just code contributions. They are the first time most developers will see confidence-aware extraction in a working example. An agent that checks the confidence score on every field and routes accordingly. That pattern does not exist in any framework cookbook yet.

The medium-term build is the access readiness dimension. ShopGraph already tracks whether target URLs require agent authentication, but the dimension is stubbed at weight 0.00 in the scoring system. It activates when Web Bot Auth adoption reaches detection threshold. When more than 10% of the test corpus requires cryptographic identity, access readiness becomes a scored dimension and the weights redistribute.

The longer-term build is CommerceFlow, a multi-step agent orchestration layer where extraction, scoring, and purchasing happen in a coordinated workflow. Stripe MPP handles the payment. ShopGraph handles the data and identity. The agent handles the decision logic. The human handles the exceptions.
